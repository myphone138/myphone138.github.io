一、初始化配置阶段（新环境/新项目起步）

首次使用Git、换电脑、新建项目时，先做好「身份认证」和「仓库基础设置」

1、配置全局用户名

作用说明：配置所有仓库默认身份，避免用他人账号提交
使用场景：首次安装Git后，或需要统一所有仓库提交身份时
git config --global user.name "myphone138"

2、配置全局邮箱

作用说明：配置所有仓库默认提交邮箱，需与GitHub/GitLab绑定
使用场景：首次安装Git后，确保提交能被远程仓库识别身份
git config --global user.email "myphone138@outlook.com"

3、查看当前仓库用户名

作用说明：查看当前仓库的局部用户名配置（未单独设置则显示全局）
使用场景：新仓库开发前，确认当前身份是否正确
git config user.name

4、设置当前仓库用户名

作用说明：为当前仓库单独设置用户名，覆盖全局配置
使用场景：特定仓库需用不同账号（如工作号/个人号区分）时
git config user.name "myphone138"

5、初始化本地仓库

作用说明：在当前文件夹创建.git隐藏目录，将文件夹变为Git可管理仓库
使用场景：新建本地项目，首次纳入Git版本控制时
git init

6、关联远程仓库

作用说明：将本地仓库与远程仓库（如GitHub/GitLab）建立关联
使用场景：本地项目需同步到远程，首次关联远程仓库时
git remote add origin git@github.com:myphone138/myphone138.github.io.git

7、检查远程仓库地址

作用说明：显示当前仓库关联的远程仓库地址（包括fetch和push地址）
使用场景：关联远程后，确认地址是否正确，避免推送到错误仓库
git remote -v

二、日常开发阶段（写代码时的高频操作）

每天写代码、改bug时，反复用到的「本地代码管理」命令

1、拉取远程最新代码

作用说明：从远程仓库拉取最新代码到本地，同步团队其他人的修改
使用场景：每天开始开发前，避免本地代码与远程脱节导致冲突
git pull

2、切换开发分支

作用说明：切换到指定的开发分支（如dev分支），避免直接修改主分支
使用场景：需在专属分支开发功能/修复bug时，如从main切换到dev分支
git checkout <分支名>

3、查看工作区修改

作用说明：查看当前工作区（未暂存）与暂存区的代码差异
使用场景：写完一段代码后，检查自己具体修改了哪些内容，避免误改
git diff

4、添加修改到暂存区

作用说明：将工作区的指定修改添加到暂存区，准备提交到本地仓库
使用场景：确认修改无误后，将需要提交的内容暂存（可指定单个文件）
git add <文件名>

5、查看暂存区差异

作用说明：查看暂存区（已add）与本地仓库（已commit）的代码差异
使用场景：暂存修改后，再次确认暂存内容是否符合预期，再提交
git diff --staged

6、提交到本地仓库

作用说明：将暂存区的内容提交到本地仓库，附带修改说明（备注需清晰）
使用场景：完成一个小功能/修复一个bug后，保存当前代码状态
git commit -m "备注（如：修复登录按钮点击无响应bug）"

7、查看提交历史

作用说明：查看当前分支的所有提交记录，包括提交人、时间、备注
使用场景：回顾自己的修改记录，或查找某个历史版本的提交信息
git log

8、回退到指定版本

作用说明：切换到指定的历史版本，恢复代码到之前可正常运行的状态
使用场景：代码改崩后，需回退到之前的稳定版本（版本号从git log获取）
git checkout <版本号>

三、协作同步阶段（和团队/远程仓库交互）

完成本地开发后，同步到远程、或处理团队协作的命令

1、再次拉取远程代码

作用说明：推送前再次拉取远程最新代码，避免与他人修改冲突
使用场景：本地提交后、推送远程前，确保本地代码是最新的
git pull

2、推送本地提交到远程

作用说明：将本地仓库的提交推送到已关联的远程仓库（默认origin）
使用场景：本地完成开发后，同步修改到远程，让团队其他人可见
git push

3、推送指定分支到远程

作用说明：将本地指定分支推送到远程仓库的对应分支
使用场景：开发新功能分支（如feature/login）后，推送分支到远程审核
git push origin <分支名>

4、查看关联远程仓库

作用说明：列出当前仓库已配置的所有远程仓库名称（默认通常是origin）
使用场景：多远程仓库（如同时关联GitHub和Gitee）时，确认可用远程
git remote

5、修改远程仓库地址

作用说明：修改指定远程仓库的连接地址（如切换HTTP/SSH协议，或远程仓库迁移）
使用场景：远程仓库地址变更，或想从HTTP协议改用更安全的SSH协议
git remote set-url origin <新远程地址>

6、检查远程连接有效性

作用说明：列出远程仓库的引用（分支、标签），验证本地能否正常连接远程
使用场景：推送失败时，排查是否远程地址错误、网络不通或权限不足
git ls-remote origin

7、测试GitHub SSH连接

作用说明：测试与GitHub的SSH连接是否正常，验证SSH密钥配置是否正确
使用场景：首次配置SSH密钥后，或SSH连接出现问题时，验证连接状态
ssh -T git@github.com

四、进阶管理阶段（安全加固/复杂场景）

需身份验证、强制规范的场景（如开源项目、企业开发）

1、查看本地GPG密钥

作用说明：列出本地已生成的GPG密钥，获取用于签名的密钥ID
使用场景：准备配置GPG签名前，确认本地有可用密钥
gpg --list-secret-keys

2、设置默认GPG签名密钥

作用说明：配置全局/本地默认的GPG签名密钥，无需每次提交指定密钥
使用场景：需频繁进行GPG签名提交时，简化操作（密钥ID从上方命令获取）
git config [--global] user.signingkey C2939BD52F9F32A304FE3E6027F8B13D87C451E1

3、GPG签名提交

作用说明：提交时附加GPG签名，证明提交是本人操作，增强安全性
使用场景：开源项目贡献、企业安全要求高的场景，需验证提交者身份
git commit -S -m "备注（如：添加用户注册功能，已签名）"

4、开启默认自动签名

作用说明：开启全局/本地默认提交签名，所有提交自动附加GPG签名（无需加-S）
使用场景：项目强制要求所有提交必须签名时，避免遗漏签名步骤
git config [--global] commit.gpgsign true

5、查看提交签名信息

作用说明：查看提交历史中的GPG签名详情，包括签名是否有效、签名者信息
使用场景：审核他人提交时，验证签名是否有效，确认提交身份真实性
git log --show-signature

6、对比两个分支差异

作用说明：查看两个指定分支之间的代码差异，包括新增、删除、修改的内容
使用场景：合并分支前，检查两个分支的差异，提前发现潜在冲突风险
git diff <分支1> <分支2>

7、生成4096位SSH密钥

作用说明：生成高强度的4096位RSA SSH密钥对，用于安全连接远程仓库
使用场景：首次配置SSH访问GitHub/GitLab等远程仓库时
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

8、查看SSH密钥指纹

作用说明：显示SSH公钥的指纹信息，用于验证密钥唯一性
使用场景：确认SSH密钥是否正确生成，或在远程仓库中识别密钥
ssh-keygen -lf ~/.ssh/id_rsa.pub

9、生成4096位GPG密钥

作用说明：生成高强度的4096位RSA GPG密钥对，用于提交签名
使用场景：首次配置GPG签名提交时
gpg --full-generate-key

10、查看GPG密钥详细信息

作用说明：显示GPG密钥的详细信息，包括ID、创建时间等
使用场景：获取GPG密钥ID用于Git配置，或查看密钥创建时间
gpg --list-secret-keys --keyid-format LONG

11、查看未导入GPG密钥文件信息

作用说明：查看未导入的GPG密钥文件的详细信息，包括ID和创建时间
使用场景：在导入GPG密钥前，确认密钥文件的有效性和信息
gpg --with-fingerprint ./path/to/key.asc

五、分支管理阶段（开发流程核心）

分支是Git的核心功能之一，用于并行开发、功能隔离和版本管理

1、查看本地分支

作用说明：列出所有本地分支，并用星号标示当前所在分支
使用场景：确认当前所在分支，查看本地已有的分支列表
git branch

2、创建并切换到新分支

作用说明：创建一个新分支并立即切换到该分支
使用场景：开始新功能开发或修复bug时，创建独立的开发分支
git checkout -b <分支名>

3、切换分支

作用说明：切换到指定分支
使用场景：在不同功能分支之间切换，或切换到主分支进行合并
git checkout <分支名>

4、查看所有分支（包括远程）

作用说明：列出所有本地和远程分支
使用场景：查看远程分支状态，了解团队其他成员的工作进展
git branch -a

5、合并分支

作用说明：将指定分支合并到当前分支
使用场景：功能开发完成后，将功能分支合并到主分支
git merge <分支名>

6、删除分支

作用说明：删除指定的本地分支
使用场景：功能已合并到主分支后，删除已完成使命的功能分支
git branch -d <分支名>

7、推送到远程分支

作用说明：将本地分支推送到远程仓库的同名分支
使用场景：将本地创建的分支推送到远程，便于团队协作或备份
git push origin <分支名>
